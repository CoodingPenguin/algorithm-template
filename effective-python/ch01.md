### Better Way 2. PEP8 스타일 가이드를 따르라

- 원본: https://www.python.org/dev/peps/pep-0008/
- 해석: https://wikidocs.net/7896

#### 공백

- `Tab` 대신 `Spacebar` 4개로 들여쓴다.
- 한 줄은 79개의 문자 이하로 쓴다.
- 긴 식을 다음 줄에 이어서 쓸 경우 `Spacebar` 4개 들여쓴 후 이어쓴다.
- 파일 안에 함수와 클래스 사이에는 빈 줄 2줄 추가한다.
- 클래스 안에 메서드 사이에는 빈 줄 1줄 추가한다.
- 딕셔너리는 `key: value`로 작성한다.
- type hint는 `param: datatype`으로 작성한다.

#### 명명규약

- 이름

```python
# 함수, 변수, 속성
lowercase_underscore

# protected 인스턴스 속성
_leading_underscore

# private 인스턴스 속성
__leading_underscore

# 클래스, 예외
CapitalizedWord

# 상수
ALL_CAPS
```

- 호출 대상 객체

```python
class Something:
    count = 0

    # 항상 self로 통일
    def __init__(self):
        print("Something")

    def do_something(self, thing):
        print(f"Do {thing}")

    # 클래스메서드의 경우만 cls로 ㅍ시
    @classmethod
    def count(cls):
        print(f"Total count is {cls.count}")
```

#### 식과 문

- 긍정적인 식을 부정하지 말고 부정을 내부에 넣어라.

```python
# 큰 범위에서 not을 쓰지 말고
if not a is b:
    pass

# 내부에 넣어라
if a is not b:
    pass
```

- 빈 문자열과 빈 리스트는 `False`로, 그렇지 않은 문자열과 리스트는 `True`로 취급된다는 것을 알고 써라. (`len`을 사용하여 검사하지 말 것!)

```python
# 비어있는지 검사
if not lst:
    pass
# 요소가 있는지 검사
if lst:
    pass
```

- if, while, for, except문을 한 줄로 줄여쓰지 말아라.

```python
# 짧더라도 이렇게 쓰지 말 것
if is_true: return True
```

#### 임포트

- import문을 항상 파일 맨 앞에 위치시킨다.
- 모듈을 임포트할 때는 절대적인 이름을 사용한다.

```python
import mypkg.sibling
from mypkg import sibling
from mypkg.sibling import example
```

- 상대적인 경로로 임포트해야하는 경우 다음과 같은 명시적인 구문을 사용한다.

```python
from . import sibling
from .sibling import example
```

- 임포트시 `표준 라이브러리` - `서드 파티 모듈` - `내가 만든 모듈`로 섹션을 나누고 각 섹션은 알파벳 순으로 정렬하라.
